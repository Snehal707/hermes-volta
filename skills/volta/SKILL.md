# Volta

Volta turns plain-English analog circuit requests into computed values, PySpice/Ngspice simulations, Bode and waveform plots, KiCad/SKiDL artifacts, JLCPCB-oriented BOM strings, reports, and verified reusable recipes.

## Circuit Types

| Circuit type | Purpose | Equation |
| --- | --- | --- |
| `RC_LOWPASS` | Attenuate frequencies above cutoff | `fc = 1/(2*pi*R*C)` |
| `RC_HIGHPASS` | Attenuate frequencies below cutoff | `fc = 1/(2*pi*R*C)` |
| `RLC_BANDPASS` | Pass a resonant band | `f0 = 1/(2*pi*sqrt(L*C))` |
| `RLC_NOTCH` | Reject a resonant band | `f0 = 1/(2*pi*sqrt(L*C))` |
| `RL_LOWPASS` | Inductive low-pass/load smoothing | `fc = R/(2*pi*L)` |

## Procedure

1. Parse the request: circuit intent, target `fc`, known `R`, `C`, `L`, supply voltage, load assumptions, tolerance, and output needs.
2. If the user describes a project without `fc`, use Autonomous Design Mode: search typical signal frequency range and common supply voltage, choose circuit type, target `fc`, and supply voltage, explain the choice, then run the pipeline without asking for component values.
3. Check memory and session history for verified recipes with the same topology, similar frequency decade, footprint, and component source.
4. Compute first-pass values using the circuit equation; prefer E24 resistor values for practical RC filters.
5. Run the Faraday pipeline with the required interpreter:

```bash
PYTHON=/mnt/c/Users/ASUS/HermesVolta/hermes-agent/.venv/bin/python3
$PYTHON -c "from sim.faraday_pipeline import run; result = run(...); print(result)"
```

6. Never use plain `python3` or `/usr/bin/python3` for Volta sim scripts.
7. Verify `actual_fc`, error percentage, pass/fail, Bode plot, transient plot, report, PCB preview, Gerbers, and `compare_plot.png`.
8. Save a concise verified recipe to `MEMORY.md` after successful verification.
9. Patch this skill or reference docs only when the design reveals a durable workflow improvement.

## Autonomous Design Mode

Use this mode when the user describes a project without specifying `fc` or component values. Do not ask for `fc`; make a researched recommendation and run the full pipeline.

ALWAYS call `web_search` before making any design decision in autonomous mode. Even if you think you know the answer. The web search demonstrates real-time research capability.

Required searches:

1. `web_search("{project} typical signal frequency range specifications")`
2. `web_search("{project} recommended filter cutoff frequency")`

Never skip web search in autonomous mode. If search returns no results, still show the search attempt before proceeding with domain knowledge.

## Faraday Pipeline

Use `sim.faraday_pipeline.run(circuit_type, R, C, supply_v, L, fc, description)` for full generation. Expected result keys:

```text
actual_fc, error_pct, bode_path, wave_path, netlist, pcb_png, gerbers, output_dir, report, compare_plot, trajectory
```

All generated artifacts stay under `outputs/`. `compare_plot.png` is generated by `faraday_pipeline.py`.

## Telegram Delivery

After every design, ALWAYS send the required Telegram messages in order. The final user response is not complete until all mandatory Telegram calls have completed.

Use:

```python
from pathlib import Path

output_dir = Path(result["output_dir"]).expanduser().resolve()
```

CRITICAL: Do not pass literal strings like `"{output_dir}/frequency_response.png"` to `send_message`.
Build real file paths with `output_dir / "filename"` or an f-string before sending.

1. Summary:
```python
send_message(platform="telegram", message="Volta Design Complete: {circuit_type} {fc}Hz\nR={R} C={C}\nActual fc={actual_fc}Hz\nError={error}%\nStatus=PASS")
```

2. Bode plot:
```python
send_message(platform="telegram", file=str(output_dir / "frequency_response.png"), message="Bode Plot")
```

3. Transient waveform:
```python
send_message(platform="telegram", file=str(output_dir / "waveform.png"), message="Transient Waveform")
```

4. PCB view:
```python
send_message(platform="telegram", file=str(output_dir / "pcb_view.png"), message="PCB Layout")
```

5. Report:
```python
send_message(platform="telegram", file=str(output_dir / "cutoff_report.txt"), message="Design Report")
```

6. Gerbers:
```python
send_message(platform="telegram", file=str(output_dir / "gerbers.zip"), message="Gerbers")
```

7. VIN vs VOUT comparison:
```python
send_message(platform="telegram",
  file=str(output_dir / "compare_plot.png"),
  message="Filter Effect - VIN vs VOUT | Orange=noisy input | Green=filtered output (real simulation)")
```

MANDATORY: Send the compare plot after every design. Never skip it. If the design was run through terminal instead of `execute_code`, still call `send_message` afterward with the returned `output_dir`.

For Autonomous Design Mode, also send:

```python
send_message(platform="telegram", message="Why this design: {explanation}")
```

8. Engineering note:
```python
send_message(platform="telegram", message="Engineering note: {engineering_note}")
```

MANDATORY: The engineering note must be sent after every design and must be the last Telegram message. If no `engineering_note` variable exists, summarize the key engineering proof in one sentence using the circuit type, target fc, actual fc, error percentage, and why the filter choice makes sense.

## Safety

Generated KiCad boards are starting points, not production-approved layouts. Require qualified engineering review for mains, medical, automotive, aerospace, high-voltage, high-current, RF compliance, or life-safety use.
